# Rule ‚Äî Build & Release

## Local build

Go and the Wails CLI are installed but **not on the bash tool's default PATH**:

- Go: `C:\Program Files\Go\bin` ‚Üí in bash: `/c/Program Files/Go/bin`
- Wails CLI: `C:\Users\–ê–¥–º–∏–Ω–∏—Å—Ç—Ä–∞—Ç–æ—Ä\go\bin\wails.exe`

So the canonical local build command is:

```bash
cd /e/claude/GameSaver && export PATH="/c/Program Files/Go/bin:$PATH" \
  && /c/Users/–ê–¥–º–∏–Ω–∏—Å—Ç—Ä–∞—Ç–æ—Ä/go/bin/wails.exe build 2>&1 | tail -8; echo "build: $?"
```

- A green build prints `Built 'E:\claude\GameSaver\build\bin\GameSaver.exe'` and `build: 0`.
- **Always build before committing.** A failing TS compile (`tsc -b`) fails the whole `wails build`.
- `wails build` regenerates `frontend/wailsjs/` bindings ‚Äî adding a new exported `App` method shows up there. Commit the regenerated bindings.

## Release flow (tag ‚Üí CI ‚Üí release)

There is no installer and no staging. **Every shipped change is a new tag.**

```bash
git add <files> && git commit -m "<subject>\n\n<why>\n\nCo-Authored-By: ..."
git push origin main
git tag vX.Y.Z
git push origin vX.Y.Z
```

- `.github/workflows/release.yml` triggers on `push` of tags matching `v*`.
- CI: Go 1.23 + Node 20 ‚Üí `wails build -trimpath -ldflags "..."` ‚Üí package `.zip` ‚Üí build watchdog ‚Üí publish GitHub Release with `generate_release_notes`.
- **Watch the run to conclusion** before telling the user it's shipped (see [`../protocol/build-verify-ship.md`](../protocol/build-verify-ship.md)). A typical run is ~3‚Äì8 min.

### Versioning convention used so far

- `0.x.y` ‚Äî pre-1.0. **Patch** (`z`) for fixes & small features; **minor** (`y`) bumps for big feature categories (0.4.0 controller, 0.5.0 PS shell UI, 0.6.0 monitor picker, 0.7.0 power menu).

## ldflags (what gets injected at build time)

```
-X main.AppVersion=$tag
-X 'GameSaver/internal/meta.DefaultSteamGridDBKey=$STEAMGRIDDB_KEY'   # only if the CI secret is set
```

- `main.AppVersion` defaults to `"dev"` for local builds; the self-updater skips the version check when it's `"dev"`.
- The SteamGridDB key comes from the `STEAMGRIDDB_KEY` **repo secret**, never from source. See [`secrets.md`](secrets.md).

## Release assets (per tag)

| Asset | Built how | Purpose |
|---|---|---|
| `GameSaver-vX.Y.Z-windows-amd64.zip` | `wails build` (CGO_ENABLED=1) ‚Üí zip of `GameSaver.exe` | the app; what the self-updater downloads |
| `gamesaver-watchdog-windows-amd64.exe` | `go build` CGO_ENABLED=0, `-s -w -H windowsgui` | shell-mode supervisor; downloaded on demand |
| `checksums.txt` | SHA-256 of all three | the watchdog download is verified against this |

## Icons

- `*.ico` and `*.exe` are gitignored. Icons are (re)generated by `go run ./cmd/gen-icon`, which writes `build/appicon.png` and the multi-res `internal/tray/icon.ico` (force-added / embedded via `go:embed`).
- Windows caches `build/windows/icon.ico`; `gen-icon` deletes the cached copy so Wails regenerates it. Incident: stale icon survived rebuilds.

## Don't

- ‚ùå Don't `--amend` or force-push a pushed commit/tag. New commit, new tag.
- ‚ùå Don't skip the local `wails build` "to save time" ‚Äî CI is slower feedback.
- ‚ùå Don't add a new `App` method and forget that CI regenerates bindings ‚Äî commit them so the diff is honest.
